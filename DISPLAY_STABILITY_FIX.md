# Version 3.1 表示安定化の修正内容

## 表示崩れの主な原因

### 1. 新しいHTMLと古いCSS・JavaScriptが混在する可能性

以前の版と同じ`style.css`・`site.js`を使い続けながら、Netlifyで1年間変更されないファイルとしてキャッシュしていました。この状態では、ページ本文だけ新しくなり、デザインファイルが古いまま残ることがあります。

### 2. ブラウザの保存機能が利用できない場合に処理が停止

Cookie同意状態の確認で`localStorage`へ直接アクセスしていました。プライバシー設定やプレビュー環境によってアクセスが拒否されると、メニュー、問い合わせリンク、診断、アニメーションなどの処理が途中で停止する可能性がありました。

### 3. JavaScriptが停止すると写真やカードが非表示のまま残る構造

スクロールアニメーション対象を最初から透明にしていたため、JavaScriptに問題が起きると、広い空白だけが表示される可能性がありました。

### 4. プロフィール写真の高さ

画像全体の基本設定に`height:auto`がなく、スマートフォンでプロフィール写真が必要以上に縦へ伸びる場合がありました。

## 修正内容

- `style.v3.1.css`、`site.v3.1.js`、`config.v3.1.js`へ変更
- Netlifyのキャッシュを再検証する設定へ変更
- 保存機能へのアクセスを例外処理で保護
- 写真と本文を初期状態で表示する安全な方式へ変更
- アニメーションに強制表示の予備処理を追加
- 画像へ`height:auto`を追加
- 320px幅までヘッダー、言語選択、Cookie表示を調整
- 長い翻訳文がレイアウトを押し広げないよう折り返し設定を追加
