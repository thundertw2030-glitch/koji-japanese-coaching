# Version 3.1 全体表示・機能検査レポート

## 確認した主な原因

### 1. 新旧CSS・JavaScriptの混在

Version 3.0では`style.css`と`site.js`のファイル名を以前の版から引き継いだ一方、Netlifyで`immutable`かつ1年間のキャッシュを指定していました。そのため、新しいHTMLが古いCSSやJavaScriptを読み込む可能性があり、レイアウトが大きく崩れる状態でした。

### 2. ブラウザの保存機能制限によるJavaScript停止

`localStorage`へ直接アクセスしていたため、埋め込み表示、厳格なプライバシー設定、一部のプレビュー環境などでは例外が発生し、その後のメニュー、問い合わせリンク、アニメーション、診断機能が動かなくなる可能性がありました。

### 3. 写真の高さが固定値の影響を受ける問題

グローバル画像設定に`height:auto`がなく、プロフィール画像のHTML寸法がスマートフォン表示へ影響し、画像が必要以上に縦長になる場合がありました。

## 実施した修正

- `style.v3.1.css`
- `site.v3.1.js`
- `config.v3.1.js`

へ変更し、旧キャッシュとの衝突を解消しました。

さらに以下を実施しました。

- Netlifyのキャッシュ設定を再検証型へ変更
- `localStorage`アクセスを例外処理で保護
- 本文・写真・カードを初期状態では表示する方式へ変更
- スクロールアニメーションに2.4秒の表示保証を追加
- 画像へ`height:auto`を追加
- 320px幅向けのヘッダー、見出し、Cookie表示を調整
- 長い翻訳文の折り返しとグリッド縮小を補強
- GitHub ActionsをVersion 3.1のファイル名へ更新
